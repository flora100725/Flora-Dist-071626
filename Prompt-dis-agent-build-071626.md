## 3. 資料處理管線與核心核心運算邏輯 (Data Pipeline)

在兩份檔案讀入記憶體後，系統將自動啟動以下隱藏的資料處理管線：

```
[許可證母體 (MD License)] ─── (LEFT JOIN) ───> [UDID 註冊明細 (UDID)]
        │                                             │
        ▼                                             ▼
  【特徵工程與轉換】                              【指標聚合】
  1. 許可證類型萃取 (輸/國產)                  1. 計算每張許可證註冊DI數
  2. 到期日標準化 (YYYY-MM-DD)                 2. 標記完全未登載之廠商
  3. 次類別代碼/名稱拆分

```

### 3.1 關鍵計算公式與狀態標記

* **已完成 UDI 資料登載之許可證比例**：

$$\text{登載比例} = \frac{\text{已在 UDID 表中出現至少一次的唯一許可證字號數}}{\text{許可證母體表中的唯一許可證字號總數}} \times 100\%$$


* **合規狀態分類（特徵工程標籤）**：
1. `完全合規`：許可證在效期內，且已登載基本 DI。
2. `應登未登`：許可證屬於法規已強制實施之級數（如 3 級或特定 2 級），但 UDID 表中完全查無資料。
3. `資料漏登`：有許可證字號，但 `基本 DI` 欄位為空值或格式異常。
4. `許可證已過期`：有效日期小於當前系統日期。

## 4. 三大核心分析維度規格說明

網頁主介面將透過 **分頁組件 (`st.tabs`)** 將分析切換為三大核心維度，確保資料呈現條理分明。

### 4.1 維度一：醫療器材商分析 (License Holder Insights)

此維度聚焦於「法人與機構」的履約合規表現，協助食藥署或大廠管理階層找出「邊緣化」或「嚴重漏報」的廠商。

* **關鍵指標卡 (KPI Metrics)**：
* 已完成資料登載的醫療器材商總數（以獨立統編/名稱計數）。
* 全國醫療器材商平均許可證 UDI 覆蓋率。


* **異常名單動態表格 (Data Inspection Table)**：
* **尚未完成任何一筆資料登載之許可證持有醫療器材商清單**：列出廠商名稱、統編、擁有的總許可證數。此表格內建排序功能，讓使用者一眼看出「手握最多許可證卻一筆 UDI 都沒登載」的黑名單榜首。
* **部分登載/高風險器材商名單**：擁有許可證多，但登載比例低於 50% 的廠商。


* **擴充分析 (And More...)**：
* 醫療器材商規模分群：依廠商持有的許可證數量（大型、中型、小型），交叉分析不同規模廠商的合規積極度。



### 4.2 維度二：許可證分析 (MD License Insights)

此維度以「產品通行證」為核心，檢視市場上流暢產品的整體合規地圖。

* **合規比例交叉矩陣**：
* **各等級（1、2、3 級）許可證完成資料登載比例**：以橫向堆疊條形圖呈現。例如，3 級許可證的已登載比例是否如預期達到 95% 以上，1 級是否因為法規時程較後而比例較低。


* **未完成資料登載許可證清單**：
* 提供詳細的資料表格，顯示「許可證字號」、「中文品名」、「醫療器材級數」、「有效日期」與「申請商名稱」。
* **法規階段警示燈**：根據級數與有效日期，自動在表格內標註「已逾期強制日（紅燈）」或「寬限期內（黃燈）」。



### 4.3 維度三：UDI 登載資料分析 (UDI Registration Deep-Dive)

此維度專注於「已註冊的 60 萬筆大數據」內部的分布結構，分析市場結構與發碼機構的生態。

* **廠商屬性與市佔率分析**：
* **廠商數（輸入 / 國產）**：從許可證字號（如「輸」、「陸輸」、「製」）自動分類，呈現輸入醫器與台灣國產醫器在 UDI 登載上的數量對比與比例。
* **發碼機構市佔分佈**：`GS1` 與 `HIBCC` 在台灣市場的佔有率與板塊分佈。


* **多維度登載筆數統計**：
* **各等級（1、2、3 級）UDI 登載總筆數**。
* **各次類別（Category）UDI 登載筆數**：呈現前 20 大熱門醫器次類別（如骨內植體、人工水晶體、一般手術器械）的登載總量。
