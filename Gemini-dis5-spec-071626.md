1. 執行摘要與系統願景 (Executive Summary & System Vision)
在當代醫療器材監管體制下，第三等級（Class III）高風險植入式醫療器材（以「矽膠充填之乳房彌補物，俗稱隆乳義乳」為例）的全生命週期追溯，是保障國民健康的重中之重。中華民國衛生福利部食品藥物管理署（TFDA）依據《醫療器材管理法》第十九條，強制要求相關業者與醫療機構定期申報高風險醫材之「來源」與「流向」。然而，傳統的申報系統僅具備靜態資料儲存與基礎表單稽核功能，面對繁雜且具有高度偽裝性的非對稱性漏報、黑市平行輸入（水貨）、以及非法地下診所（密醫）的洗貨交易，傳統稽查機制在反應速度與分析深度上顯得捉襟見肘。
本系統 「醫材來源流向大數據與後市場智慧稽查平台 (Medical Device Traceability and Post-Market Intelligence Audit System, 簡稱 MD-TPIAS Agentic OS)」 旨在打造一個具備自主意識（Autonomous Agency）的 「智慧 Agent 運作系統 (Agentic OS)」。本平台深度整合 Google ADK (Agent Development Kit) 的兩大核心能力：Google 搜尋工具 (google_search) 與 內建代碼執行引擎 (built_in_code_execution/VertexCodeInterpreter)，並以 gemini-2.5-flash 作為預設神經中樞，構建具備自我推理、文件檢索、數據勾稽、與行政公文自動撰寫能力的數位監管執法平台。
本說明書不含實體系統程式碼之撰寫，而是一份極度詳盡、達到工業級開發標準的架構與技術規格藍圖，字數約在 4,000 至 5,000 字之間，全面剖析系統底層架構、AI 協作機制、數據流設計、視覺化儀表板、以及革命性的前瞻 AI 功能，並於篇末提出 20 道極具深度的技術與法規實務追隨問題。

3.1 網頁搜尋工具 (google_search)
在醫材管理實務中，國外原廠（如 Mentor, Allergan）一旦發布召回資訊，台灣代理商可能會有時間差上的隱瞞。本平台配置 google_search 進入 Agent 的工具箱：
工作流程：當使用者貼入或上傳一份未知來源的 skill.md 並上傳醫材出貨清單時，Agent 如果發現其中包含某特定批號，會自主啟動 google_search，查詢例如 "Mentor breast implant recall batch 2098982 FDA"。
資料處理：Agent 自動過濾 Google 搜尋回傳的 JSON 結構，抓取官方 PDF 連結或公告新聞，擷取召回原因（如：無菌完整性受損、外殼滲漏），並將此資訊作為「外部增強背景資訊」注入至稽核報告中。
引用標註格式：所有搜尋結果在 UI 介面上必須以 [Google Search Source: URL] 形式進行高亮超連結標註，確保資料的真實性與可追溯性。

系統預設以 gemini-2.5-flash 作為運算中樞，藉此獲得低延遲、高吞吐、且具備 100 萬 Token 的極大上下文視窗。同時，系統保留下拉選單，允許稽查人員切換至 gemini-2.5-pro（用於極度複雜之法規公文撰寫與深度因果推理）或 gemini-2.0-flash-exp（進行超快速即時對話測試）。

雙行並發渲染：介面分為左右對等視窗，支援獨立滾動（Sync Scroll 協同滾動開關）。

6.2 精準比對指標與演算法
對比視窗下方即時渲染一個「智能對比儀表板」，包含以下四個客觀硬體與語意指標：
耗時對比 (Execution Latency)：以毫秒 (ms) 為單位，精確計時 API 從送出至接收完畢的時間。
Token 消耗與成本計算 (Token Metric)：顯示 Input/Output Tokens，並依據官方 API 定價公式，換算成美金成本（例如 $0.00015 USD vs $0.00120 USD）。
字元級文本差異比對 (Character-Level Diff)：使用 jsdiff 演算法，將兩側輸出的法律公文進行字元對比，綠色代表配置 B 多出的條款，紅色代表配置 A 被刪除的措辭。
