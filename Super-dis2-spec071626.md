1.1 前端核心模組 (Frontend Core Modules)
React 19 + TypeScript：全面實作強型別介面（如 DistributionRecord, PurchaseRecord, ChatMessage），避免資料解析時的隱性錯誤。

1.2 後端 API 安全代理 (Backend Server Proxy)
安全性原則（No Key Exposure）：所有的 Gemini 請求皆代理至後端 /api/gemini/chat，嚴格防範 API Key 暴露於瀏覽器端，完美契合安全設計方針。
Lazy Initialization（延遲初始化）：對於 GoogleGenAI SDK 進行延遲載入檢查，若環境變數中缺失 GEMINI_API_KEY，將回傳友好的行政引導提示，而不會造成後端容器啟動時崩潰。
User-Agent 遙測宣告：後端 SDK 請求中，httpOptions 內建帶有 'aistudio-build' 的 User-Agent 標頭，以供官方系統完整遙測追蹤。

3. 核心大數據功能 (Core Big Data Architecture)
3.1 快速回收與定向鎖定 (Quick Recall & Target)
實時追溯機制：當使用者輸入特定瑕疵批號（例如：國際通報原廠製程防漏層不均勻之 “曼陀” 批號 2098982）時，勾稽引擎會在微秒內篩選出該批號銷往的所有醫療院所。
流向狀態判定：自動歸類為「在院待用」、「已調撥」、「在庫封存」、「已運回」或「待運回」，精準指導地方衛生局稽查員直接到現場強制查扣封存。
3.2 產品生命週期與效期預警 (Lifecycle & Expiry Alerts)
動態生命週期計算法：系統以 2026/07/16 為大數據解算基準日。對比產品的 保存期限，自動精算剩餘生命週期。
多級警戒安全徽章：
🔴 違規缺失：保存期限欄位空白或格式錯誤。
💀 禁人體植入：保存期限小於當前基準日（即已過期醫材）。
🚨 極危即期：剩餘效期小於 1 年，列為強制封存防範對象。
⚠️ 黃色警戒：剩餘效期 1 至 5 年，列為常態追蹤對象。
🟢 安全良好：保存期限長，品質安全無虞。
3.3 追溯一致性雙向勾稽 (Traceability Audit)
雙向穿透算法：
完全匹配 (Perfect Match)：進口商銷售與診所採購，在產品序號、批號、數量及有效期限上完全一致。
醫院漏報 (Asymmetry Gap)：進口商有申報出貨，但醫院診所未申報購入。懷疑有隱匿非法黑市交易。
查無進口源 (Illegal Smuggling)：診所登載了購入與 UDI 掃描，但進口商卻查無任何發貨或進口紀錄。高度存在「走私水貨」、「非法仿冒仿製假貨」之重大違法嫌疑。
罰則裁量指引：針對不同結論，自動匹配《醫療器材管理法》之罰則，直接編寫處置公文草案。
